# 违章视频上传功能说明

## 📌 功能概述

为违章记录添加了视频上传和查看功能，管理员可以在录入违章信息时上传视频证据，查询违章记录时可以在小窗中播放观看视频。

---

## 🚀 部署步骤

### 1. 更新数据库结构

运行SQL脚本添加视频字段：

```bash
mysql -u root -p bus_safety_system < add_video_field.sql
```

或者手动执行：

```sql
USE bus_safety_system;
ALTER TABLE violation_record 
ADD COLUMN video_url VARCHAR(500) COMMENT '违章视频URL' AFTER evidence_url;
```

### 2. 创建上传目录

上传目录会自动创建，路径为：
```
static/uploads/videos/
```

### 3. 重启服务器

```bash
# 使用批处理文件
restart.bat

# 或者手动重启
stop.bat
start.bat
```

## 💡 功能说明

### 1. 上传视频

**位置**：录入违章信息页面

**步骤**：
1. 填写违章基本信息（司机、车辆、类型等）
2. 点击"📹 选择视频"按钮
3. 选择视频文件（支持的格式见下方）
4. 等待上传完成，会显示视频预览
5. 可以点击"删除视频"取消上传
6. 提交表单保存违章记录

**支持的视频格式**：
- MP4
- AVI
- MOV
- WMV
- FLV
- MKV
- WebM

**文件大小限制**：最大 100MB

### 2. 查看视频

**位置**：查询司机违章详情页面

**步骤**：
1. 查询司机的违章记录
2. 在结果表格的"视频"列
3. 如果有视频，会显示"📹 查看视频"按钮
4. 点击按钮在弹窗中播放视频
5. 点击关闭按钮或背景区域关闭弹窗

## 🎯 技术实现

### 后端API

#### 1. 视频上传API
```
POST /api/upload-video
Content-Type: multipart/form-data

参数：
- video: 视频文件

返回：
{
    "success": true,
    "message": "视频上传成功",
    "video_url": "/static/uploads/videos/violation_20251204_143022_a1b2c3d4.mp4",
    "filename": "violation_20251204_143022_a1b2c3d4.mp4"
}
```

#### 2. 添加违章记录API（已更新）
```
POST /api/violation
Content-Type: application/json

数据中新增字段：
- video_url: 视频URL（可选）
```

#### 3. 查询违章记录API（已更新）
```
GET /api/violations/by-driver?driver_id=xxx&start_date=xxx&end_date=xxx

返回数据中新增字段：
- video_url: 视频URL
```

### 前端功能

#### 1. 视频上传处理
- 文件选择验证（类型、大小）
- 异步上传到服务器
- 实时预览功能
- 上传进度提示

#### 2. 视频查看弹窗
- 模态窗口显示
- 自动播放视频
- 响应式设计
- 点击背景关闭

### 文件存储

**路径规则**：
```
static/uploads/videos/violation_{时间戳}_{随机码}.{扩展名}

示例：
violation_20251204_143022_a1b2c3d4.mp4
```

**命名说明**：
- `violation_`: 前缀标识
- `20251204_143022`: 上传时间
- `a1b2c3d4`: 8位随机字符
- `.mp4`: 原文件扩展名

## 📋 使用示例

### 场景1：录入带视频的违章记录

1. 管理员登录系统
2. 进入"录入违章信息"页面
3. 选择违章司机、车辆、类型等
4. 点击"选择视频"上传行车记录仪视频
5. 等待上传完成，确认预览
6. 填写其他信息，点击提交
7. 系统提示"违章信息录入成功！"

### 场景2：查看违章视频

1. 进入"查询司机违章详情"页面
2. 选择司机和时间范围
3. 点击"查询"
4. 在结果列表中找到有视频的记录
5. 点击"📹 查看视频"按钮
6. 在弹窗中观看视频
7. 点击关闭按钮或背景关闭弹窗

## ⚠️ 注意事项

1. **文件大小**：单个视频文件不能超过100MB
2. **存储空间**：定期清理过期的视频文件，避免占用过多磁盘空间
3. **网络流量**：视频文件较大，上传和播放会消耗较多带宽
4. **浏览器兼容**：建议使用Chrome、Edge、Firefox等现代浏览器
5. **视频格式**：MP4格式兼容性最好，推荐使用

## 🔧 故障排查

### 上传失败

**可能原因**：
1. 文件大小超过100MB
2. 文件格式不支持
3. 上传目录没有写权限
4. 网络连接中断

**解决方法**：
1. 压缩视频文件
2. 转换为支持的格式
3. 检查目录权限
4. 重新上传

### 视频无法播放

**可能原因**：
1. 视频文件损坏
2. 浏览器不支持该编码
3. 文件路径错误

**解决方法**：
1. 重新上传视频
2. 转换为MP4格式
3. 检查video_url是否正确
