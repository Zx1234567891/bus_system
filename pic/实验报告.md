# 实验任务

## 任务描述

本实验基于MySQL关系型数据库，构建一个完整的包含后台数据库、数据库访问程序以及前台网页的**公交安全管理系统**。实验涵盖从运用规范化理论设计数据库模式、实现完整性约束与索引优化，到借助DeepSeek大模型辅助生成前后端代码，并实现**AI自然语言数据库查询**功能的全流程。通过本任务，掌握数据库系统全栈设计与实现的能力，重点在于深入理解数据库规范化设计、完整性约束与性能优化，并实践利用AI工具提升开发效率。

**实现MySQL后台数据库**

数据库的有关语义如下：

- 公交公司有若干个车队，每个车队下有若干条线路；
- 公交公司有若干辆汽车，每辆车属于一条线路；
- 每个车队有一名队长，他只有管理工作，不开车；
- 每条线路有若干名司机，其中有一名路队长，除开车外，还承担管理工作；每名司机只在一条线路上开车；
- 司机开车时会产生违章，包含：闯红灯、未礼让斑马线、压线、违章停车等；
- 队长、路队长负责将司机的违章信息输入到系统，包含：司机、车辆、车队、线路、站点、时间、违章等。

**前台网页**

开发一个公交安全管理系统来对数据库进行访问，使用Python Flask Web框架，功能如下：

- 录入司机基本信息，如工号、姓名、性别等；
- 录入汽车基本信息，如车牌号、座数等；
- 录入司机的违章信息；
- 查询某个车队下的司机基本信息；
- 查询某名司机在某个时间段的违章详细信息；
- 查询某个车队在某个时间段的违章统计信息，如：2次闯红灯、4次未礼让斑马线等；
- **通过AI自然语言查询/修改数据库信息**（DeepSeek大模型驱动）。

**注意事项**

- 后台数据库和前台程序使用DeepSeek大模型辅助开发；
- 在数据库的设计过程中运用规范化理论，避免出现插入异常、删除异常、数据冗余等问题；
- 设定关系的完整性规则，如实体完整性（设置主码）、参照完整性（设置外码和对应的主码）、用户自定义完整性（如性别只能为"男"或"女"）；
- 使用索引来加快查询的速度；使用视图、函数、存储过程来简化系统的设计；
- 系统集成DeepSeek AI，实现自然语言转SQL的智能查询功能。

## 关键步骤

### 步骤一：E-R图设计与关系模式转换

#### E-R图设计

根据公交安全管理系统的业务语义，识别出以下实体及其属性：

- **员工**（employee_id, 工号, 姓名, 性别, 身份证号, 联系电话, 地址, 入职日期, 状态）
- **车队**（fleet_id, 车队编号, 车队名称, 描述）
- **线路**（route_id, 线路编号, 线路名称, 起点站, 终点站, 总里程, 票价, 状态）
- **司机**（driver_id, 驾驶证号, 驾照类型, 驾照有效期, 驾龄, 是否路队长） — 继承自员工
- **车辆**（bus_id, 车牌号, 车辆编号, 车型, 品牌, 座位数, 购置日期, 状态）
- **站点**（station_id, 站点编号, 站点名称, 地址, 经度, 纬度）
- **违章类型**（type_id, 违章代码, 违章名称, 扣分, 罚款, 严重程度）
- **违章记录**（record_id, 违章时间, 违章地点, 记录时间, 处理状态, 备注, 视频URL）

实体之间的关系：
- 车队 ←1:n→ 线路（一个车队管理多条线路）
- 车队 ←1:1→ 队长/员工（每个车队有一名队长）
- 线路 ←1:n→ 司机（每条线路有多名司机）
- 线路 ←1:n→ 车辆（每辆车属于一条线路）
- 线路 ←m:n→ 站点（一条线路有多个站点，一个站点可以属于多条线路）
- 司机 + 车辆 + 违章类型 → 违章记录（多对多联系）

#### 将E-R图转换成关系模式

**将实体转换成关系模式：**

- 员工表（**员工ID**, 工号, 姓名, 性别, 身份证号, 电话, 地址, 入职日期, 状态）
- 车队表（**车队ID**, 车队编号, 车队名称, 描述）
- 线路表（**线路ID**, 线路编号, 线路名称, 起终点站, 里程, 票价, 状态）
- 司机表（**司机ID**, 驾驶证号, 驾照类型, 有效期, 驾龄, 是否路队长）
- 车辆表（**车辆ID**, 车牌号, 车辆编号, 车型, 品牌, 座位数, 购置日期, 状态）
- 站点表（**站点ID**, 站点编号, 站点名称, 地址）
- 违章类型表（**类型ID**, 违章代码, 违章名称, 扣分, 罚款, 严重程度）

**将联系转换成关系模式：**

1:n联系 — 将"1"端的主键加入到"n"端实体对应的表中：

- 车队表（**车队ID**, 车队编号, 车队名称, *队长ID*, 描述）
- 线路表（**线路ID**, 线路编号, 线路名称, *所属车队ID*, 起终点站, 里程, 票价, 状态）
- 车辆表（**车辆ID**, 车牌号, 车辆编号, 车型, 品牌, 座位数, *所属线路ID*, 状态）
- 司机表（**司机ID**, 驾驶证号, 驾照类型, 有效期, *所属线路ID*, 驾龄, 是否路队长）

m:n联系 — 需要单独建表：

- 线路站点关联表（***线路ID***, ***站点ID***, 站序, 方向, 距起点距离）
- 违章记录表（**记录ID**, *司机ID*, *车辆ID*, *车队ID*, *线路ID*, *站点ID*, *违章类型ID*, 违章时间, 地点, *记录人ID*, 状态, 备注）

**最终关系模式（符合3NF）**

- employee（**employee_id**, emp_code, name, gender, id_card, phone, address, hire_date, status）
- fleet（**fleet_id**, fleet_code, fleet_name, *captain_id*, description）
- route（**route_id**, route_code, route_name, *fleet_id*, start_station, end_station, total_distance, ticket_price, status）
- driver（***driver_id***, license_no, license_type, license_expire_date, *route_id*, is_route_captain, driving_years）
- bus（**bus_id**, plate_number, bus_code, model, brand, seats, purchase_date, *route_id*, status）
- station（**station_id**, station_code, station_name, address, longitude, latitude）
- route_station（***route_id***, ***station_id***, sequence_no, direction, distance_from_start）
- violation_type（**type_id**, type_code, type_name, penalty_points, fine_amount, severity）
- violation_record（**record_id**, *driver_id*, *bus_id*, *fleet_id*, *route_id*, *station_id*, *violation_type_id*, violation_time, violation_location, *recorded_by*, status, remarks, video_url）

```sql
    name VARCHAR(50) NOT NULL COMMENT '姓名',
    gender ENUM('男', '女') NOT NULL COMMENT '性别',
    id_card VARCHAR(18) NOT NULL UNIQUE COMMENT '身份证号',
    phone VARCHAR(11) COMMENT '联系电话',
    address VARCHAR(200) COMMENT '家庭住址',
    hire_date DATE NOT NULL COMMENT '入职日期',
    status ENUM('在职', '离职', '休假') NOT NULL DEFAULT '在职' COMMENT '员工状态',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    -- 用户自定义完整性约束
    CONSTRAINT chk_id_card CHECK (LENGTH(id_card) = 18),
```

#### 添加外键约束（参照完整性）

```sql
-- 车队表外键：队长必须是员工
ALTER TABLE fleet ADD CONSTRAINT fk_fleet_captain
    FOREIGN KEY (captain_id) REFERENCES employee(employee_id)
    ON DELETE SET NULL ON UPDATE CASCADE;

-- 线路表外键：线路属于车队
ALTER TABLE route ADD CONSTRAINT fk_route_fleet
    FOREIGN KEY (fleet_id) REFERENCES fleet(fleet_id)
    ON DELETE RESTRICT ON UPDATE CASCADE;

-- 司机表外键
ALTER TABLE driver
    ADD CONSTRAINT fk_driver_employee
    FOREIGN KEY (driver_id) REFERENCES employee(employee_id)
    ON DELETE CASCADE ON UPDATE CASCADE,
    ADD CONSTRAINT fk_driver_route
    FOREIGN KEY (route_id) REFERENCES route(route_id)
    ON DELETE SET NULL ON UPDATE CASCADE;

-- 车辆表外键
ALTER TABLE bus ADD CONSTRAINT fk_bus_route
    FOREIGN KEY (route_id) REFERENCES route(route_id)
    ON DELETE SET NULL ON UPDATE CASCADE;

-- 违章记录表外键
ALTER TABLE violation_record
    ADD CONSTRAINT fk_vr_driver FOREIGN KEY (driver_id) REFERENCES driver(driver_id),
    ADD CONSTRAINT fk_vr_bus FOREIGN KEY (bus_id) REFERENCES bus(bus_id),
    ADD CONSTRAINT fk_vr_fleet FOREIGN KEY (fleet_id) REFERENCES fleet(fleet_id),
    ADD CONSTRAINT fk_vr_route FOREIGN KEY (route_id) REFERENCES route(route_id),
    ADD CONSTRAINT fk_vr_station FOREIGN KEY (station_id) REFERENCES station(station_id),
```

#### 创建索引（加速查询）


```python
def hash_password(password, salt=None):
    """密码哈希函数"""
    if salt is None:
        salt = secrets.token_hex(32)
    password_hash = hashlib.sha256((password + salt).encode()).hexdigest()
    return password_hash, salt

def verify_password(password, password_hash, salt):
    """验证密码"""
    computed_hash, _ = hash_password(password, salt)
    return computed_hash == password_hash

def login_required(f):
    """登录验证装饰器"""
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if 'user_id' not in session:
            return redirect(url_for('login_page'))
        return f(*args, **kwargs)
    return decorated_function
```

```python
def extract_sql_from_response(ai_response):
    """从AI回复中提取SQL语句"""
    import re
    pattern = r'```sql\s*\n?(.*?)\n?\s*```'
    matches = re.findall(pattern, ai_response, re.DOTALL | re.IGNORECASE)
    if matches:
        sql = matches[0].strip()
        if sql.endswith(';'):
            sql = sql[:-1].strip()
        return sql
    return None
```

```python
        'DROP TABLE', 'DROP DATABASE', 'TRUNCATE', 'ALTER TABLE',
        'CREATE TABLE', 'CREATE DATABASE', 'GRANT', 'REVOKE'
```

```python
            cursor.execute(sql)
            if sql.strip().upper().startswith('SELECT'):
                rows = cursor.fetchall()
                # 处理日期和Decimal序列化
                for row in rows:
                    for key, val in row.items():
                        if isinstance(val, (datetime, date)):
                            row[key] = val.isoformat()
                        elif isinstance(val, Decimal):
                            row[key] = float(val)
                return {'success': True, 'type': 'query', 'data': rows, 'count': len(rows)}
            else:
                conn.commit()
                affected = cursor.rowcount
                return {'success': True, 'type': 'modify', 'affected_rows': affected}
    except Exception as e:
        conn.rollback()
        return {'success': False, 'error': str(e)}
    finally:
        conn.close()
```

```python
def ai_chat():

    history = data.get('history', [])

    ]
    has_db_intent = any(kw in user_message for kw in db_keywords)

    if has_db_intent:
        # ===== 自然语言数据库查询模式 =====
        # Step 1: 调用DeepSeek生成SQL
        nl2sql_messages = [{'role': 'system', 'content': NL2SQL_SYSTEM_PROMPT}]
        if history:
            nl2sql_messages.extend(history[-10:])
        nl2sql_messages.append({'role': 'user', 'content': user_message})

        ai_response = call_deepseek_non_stream(nl2sql_messages)

        if sql:

                return _send_response("安全警告：检测到危险操作，已拒绝执行。")

            # Step 3: 执行SQL
            exec_result = execute_nl_sql(sql)

            # Step 4: 组合最终回复（AI解释 + 执行结果表格）
            if exec_result['success']:
                if exec_result['type'] == 'query':
                    table_str = format_query_result_as_table(exec_result['data'])

                                  str(exec_result['affected_rows']) + " 条记录。"
            else:
                result_text = ai_response + "\n\n---\n**执行出错**：" + exec_result['error']
```

```python
def _stream_chat(user_message, history, system_prompt):
    """普通聊天的流式输出"""
    messages = [{'role': 'system', 'content': system_prompt}]
    if history:
        messages.extend(history[-20:])
    messages.append({'role': 'user', 'content': user_message})

    def generate():
        import requests as req
        response = req.post(
            DEEPSEEK_API_URL,
            headers={
                'Content-Type': 'application/json',
                'Authorization': f'Bearer {DEEPSEEK_API_KEY}'
            },
            json={
                'model': 'deepseek-chat',
                'messages': messages,
                'temperature': 0.7,
                'max_tokens': 8000,
                'stream': True  # 启用流式输出
            },
            timeout=60,
            stream=True  # 流式接收
        )
        full_content = ""
        for line in response.iter_lines():
            if line:
                line = line.decode('utf-8')
                if line.startswith('data: '):
                    data_str = line[6:]
                    if data_str == '[DONE]':
                        yield f"data: {json.dumps({'done': True}, ensure_ascii=False)}\n\n"
                        break
                    try:
                        chunk_data = json.loads(data_str)
                        delta = chunk_data['choices'][0].get('delta', {})
                        content = delta.get('content', '')
                        if content:
                            full_content += content
                            yield f"data: {json.dumps({'content': content}, ensure_ascii=False)}\n\n"
                    except Exception:
                        continue

    return Response(stream_with_context(generate()), mimetype='text/event-stream')
```
